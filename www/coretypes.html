Rx: Simple, Extensible Schemata
Types and Schemata

There are really only two things you need to know about in Rx: types and schemata. A type is an abstract descripition of a type of data. For example, "an integer" or "an associative array" are data types. When they're made into concrete, checkable constraints, they become schemata. This is done by parameterizing them.

A Rx type is identified by a string, its type name. The type name is broken down into two parts, the authority and the subname. It's written like this: /authority/subname

While users can define their own custom types, quite a lot of data validation rules can be expressed using the core types that come with a basic implementation of Rx. These types use the empty string as their authority, which means that they look like this: //subname

This page gives a brief overview of the core types, which will be defined in detail elsewhere, later.

Simple Scalar Type

The basic scalar (one-value) types are:

//bool
a value that is either true or false
//num
a number; may be parameterized by a range
//int
a integer; any number such that (x == floor(x)); may be parameterized by range
//str
a string, even the empty string
//nil
the lack of a value (undef, null, nil, etc)
//def
any defined value; anything that is not //nil
//scalar
any single value (or //nil) -- anything but a collecting type

The basic collecting (many-value) types are:

//arr
a list of values all of one type; may be parameterized by length
//seq
a list of values all of varying types; each slot has a schema
//map
a mapping of string keys to values; parameterized by required and optional keys, and how to treat unknown keys
