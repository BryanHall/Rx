//str

The //str type accepts strings and only strings.

The optional value parameter may provide an exact value to be compared againt. If this paramter is given, the schema will only accept a string that is equal to the parameter's value.

/.meta/str, the schema for //str definitions:

{
  "type": "//rec",
  "required": {
    "type": { "type": "//str", "value": "//str" }
  },
  "optional": {
    "value": { "type": "//str" }
  }
}
